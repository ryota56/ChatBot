# ウェブサイト要件定義書

## 1. 機能要件

### 1.1 トップ画面
- 中央に3枚の画像をスライド形式で表示
- 画像間をスライド切り替え可能な複数のUIを実装:
  - 画像の左右に配置した矢印ボタン（[←]/[→]）でのナビゲーション
  - 下部のスライドインジケーター（○ ● ○）でのナビゲーション
  - スワイプ操作でのナビゲーション（モバイル向け）
- 画像タップでチャット画面に遷移（下にスクロール）

### 1.2 チャット機能
- Dify APIを使用したチャットボット連携
- 選択したキャラクターとチャット可能
- チャット画面では選択したキャラクターの画像が右上に表示されたまま
- 自然な会話形式のインターフェース
- Difyで設定したワークフローに基づくキャラクター固有の応答

### 1.3 API連携
- Dify APIとの連携
- チャットワークフローの実装
- APIレスポンスの適切なハンドリング

## 2. デザイン要件
- 有名ゲームを参考にした視認性の高いデザイン
- 色合いは美しく見やすいものを採用
- レスポンシブ対応（様々なデバイスサイズに対応）
- スライドナビゲーションの矢印ボタンは直感的で見やすいデザイン
- スムーズなアニメーション効果（スライド切り替え、チャット画面遷移）

## 3. ユーザーエクスペリエンス（UX）の流れ
1. **初期表示**: トップ画面でキャラクター画像を表示
2. **画像切り替え**: 矢印ボタン、インジケーター、スワイプで画像切り替え
3. **キャラクター選択**: 画像タップでチャット画面に移行
4. **チャット体験**: キャラクターとの対話、右上にキャラクター画像表示
5. **キャラクター切り替え**: 上部に戻り、別のキャラクターを選択可能

## 4. 技術的検討事項
1. Dify APIの仕様確認と連携方法
2. 画像スライダーの実装（複数ナビゲーションオプション、アニメーション効果含む）
3. スクロールトランジションの実装
4. チャットUIの設計
5. キャラクター画像の固定表示（スクロール時）
6. レスポンシブデザイン対応

## 5. 開発ステップ案
1. デザインモックアップ作成
2. フロントエンド基本構造実装
   - スライドシステム（矢印ボタン、インジケーター、スワイプ機能）
   - スクロールトランジション
3. Dify API連携実装
4. チャットインターフェース実装
5. UIアニメーション実装
6. レスポンシブ対応
7. テスト・デバッグ
